# Contributing to Grand Royal Hotel Food Delivery App

Thank you for your interest in contributing! Here are the guidelines to follow:

## Code of Conduct

- Be respectful and professional
- Avoid harassment or discrimination
- Report issues privately if needed

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/fooddelivery-app.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit: `git commit -am 'Add feature description'`
6. Push: `git push origin feature/your-feature-name`
7. Create a Pull Request

## Commit Message Guidelines

Use clear, descriptive commit messages:
- ✨ `feat:` New feature
- 🐛 `fix:` Bug fix
- 📚 `docs:` Documentation
- 🎨 `style:` Code style changes
- ♻️ `refactor:` Code refactoring
- ⚡ `perf:` Performance improvements
- ✅ `test:` Tests
- 🔧 `chore:` Build/config changes

Example: `feat: add restaurant search with filters`

## Code Style

- Follow existing code patterns
- Use consistent naming conventions
- Add comments for complex logic
- Use prettier for formatting: `npm run format`

## Testing

- Write tests for new features
- Ensure all tests pass: `npm test`
- Maintain or improve code coverage

## Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Run linter: `npm run lint`
4. Ensure CI/CD passes
5. Request review from maintainers

## Reporting Bugs

1. Check if bug already exists
2. Provide clear description
3. Include steps to reproduce
4. Add screenshots/logs if applicable

## Feature Requests

1. Check if feature already requested
2. Describe the feature clearly
3. Explain the use case and benefits
4. Provide examples if possible

## Questions?

Feel free to open a discussion or contact the team!
